Навіщо це

Для чого підходить шаблон

Шаблон створено для швидкого запуску SPA-застосунків. Замість того щоб щоразу налаштовувати однаковий фундамент, ви стартуєте з робочого застосунку, де вже є те, що потрібно більшості продуктів з першого дня.

  • Авторизація з коробки — email, телефон (SMS) та соціальні мережі, із сесіями, CSRF-захистом і тротлінгом.
  • Реалтайм за замовчуванням — налаштований WebSocket-шар для живих оновлень, presence і двостороннього обміну.
  • Push-сповіщення — серверна доставка web-push підключеним клієнтам.
  • Встановлюваний PWA — фронтенд на Vue 3 готовий до встановлення й офлайн-роботи.

Готові сценарії

Робочі продуктові флоу

  • Реєстрація email, підтвердження пошти, вхід/вихід, logout-all, зміна та скидання пароля (зокрема через one-time токен).
  • Опційна авторизація за телефоном (register / confirm / complete) та прив'язка телефону й email до акаунта.
  • OAuth redirect/callback як основа для соціального входу.
  • Сторінка акаунта з профілем та керуванням аватаром.
  • Керування підпискою на push + ендпоінт «надіслати тестовий push».
  • Адмін-зона: користувачі, онлайн-користувачі, історія онлайну, база знань.
  • AI-підтримка: чат з історією та статтями бази знань.
  • API playground для ручного тестування HTTP/WebSocket.

Безпека

Захист «з коробки» + CSP

🍪

Сесії та CSRF

Cookie-сесії в Redis, synchronizer-token CSRF із ротацією на login/logout, режим report-only до ввімкнення enforce.

🌐

CORS & proxy

Білий список origin-ів, довіра до проксі лише за TRUSTED_PROXY_CIDRS, тротлінг за реальним IP.

🛡️

Content Security Policy

Політика директивами у config/csp.ts + reportOnly для безпечного викату.

CSP як директиви, а не «магічний рядок». script-src, style-src, img-src, connect-src, frame-ancestors та інші налаштовуються в одному місці; connect-src залежить від середовища (суворіше у production, м'якше локально). Заголовок видається на HTML-відповідях зі static-сервера — опційно (вмикається в коді), розкочуйте з reportOnly: true.

Старт

Запуск за 3 кроки

# 1. Встановити залежності воркспейсу
pnpm install

# 2. Налаштувати оточення бекенду
cp packages/backend/.env.example packages/backend/.env

# 3. Зібрати shared і запустити backend + frontend
pnpm dev
Backend → :3000 Frontend → :5173 API playground → :5174

Для базового старту достатньо PostgreSQL; Redis потрібен для повної auth/realtime поведінки. ./init.sh проганяє install → typecheck → lint → test → build.

Appendix · маршрут

Опис маршруту

Групи маршрутів у src/app/routes/http-routes.ts: спільний префікс і middleware-и для всієї групи. 

// src/app/routes/http-routes.ts
{
  prefix: "avatar",
  middlewares: ["session_web", "csrf_guard", "auth_guard"],
  group: [
    defineRoute({
      url: "/upload",
      method: "post",
      handler: AvatarController.uploadAvatar.bind(AvatarController),
      allowedContentTypes: ["multipart"],   // для файлів
      validator: EmptyFormInputSchema,        // ArkType зі shared
      ResponseSchema: ResponseSchemas.UploadAvatarResponseSchema,
      description: "Upload user avatar",
    }),
  ],
}
Appendix · контролер

Контролер — лише транспорт

Типізований payload → виклик сервісу → мапінг помилок на HTTP-коди. Бізнес-логіки та SQL тут немає. 

async login(context: HttpContext<LoginInput>): Promise<LoginResponse> {
  const payload = getTypedPayload(context); // типізовано як LoginInput

  const result = await authService.login(payload, context.auth, context.session);

  if (Result.isError(result)) {
    return mapControllerError(context, result.error);
  }
  return result.value; // { status, user, wsUrl, wsToken }
}
